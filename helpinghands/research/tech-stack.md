# Tech Stack -- Helping Hands Cleaning

## MaidCentral (Operations CRM)

### What It Is
Cloud-based business management platform built exclusively for residential cleaning companies. Founded by Tom Stewart. Not a generic field service tool -- purpose-built by cleaning industry insiders.

### Capabilities
- Scheduling & dispatch (real-time availability, drag-and-drop calendar)
- Full CRM (room-by-room customer preferences, visit history, employee likes/dislikes)
- Online booking widget (embeddable iFrame, real-time pricing + availability, page-specific codes)
- Batch billing & invoicing (QuickBooks Online sync)
- Payroll reporting (Paystri integration)
- KPI tracking & financial dashboards
- Quality management (scorecards per visit, customer ratings)
- Automated campaigns (email/SMS follow-ups, review requests)
- Customer portal (`helpinghands.maidcentral.com`) -- self-service schedule, payments, tips, ratings
- Franchise/multi-location support

### API & Integrations
- **REST API** with documentation at `support.maidcentral.com/apidocs/`
- API auth: create API users in admin, POST to `/token` for Bearer token
- Full booking flow API available (custom booking experiences possible)
- Lead form POST endpoint available

## AEO Implementation Architecture (WordPress)

Syntora's own AEO pipeline uses Next.js + Supabase + Vercel. Helping Hands runs WordPress + Divi on managed hosting. The automation needs to integrate differently:

### Pipeline

```
Question Mining (Python + Reddit RSS + Gemini)
    |
    v
AEO Page Generation (Python + Gemini)
    |
    v
Quality Validation (8-check QA pipeline)
    |
    v
WordPress REST API (publish as custom post type or hidden pages)
    |
    v
AIOSEO Pro (auto-generates sitemap entries + schema markup)
    |
    v
robots.txt (must add GPTBot, PerplexityBot, ClaudeBot, GoogleOther)
    |
    v
Google Search Console (submit AEO-specific sitemap)
```

### Access Needed
- WordPress admin (REST API credentials + application password)
- AIOSEO Pro settings (to configure schema on new page templates)
- Hosting panel (H2O -- to edit robots.txt if not accessible via WP)
- Google Search Console (to submit sitemaps and monitor indexing)
- GHL admin (for lead flow integration + voice agent setup)

### WordPress Integration Details
- **WP REST API** (`/wp-json/wp/v2/`) for creating/updating pages programmatically
- **Authentication:** Application Passwords (WordPress 5.6+) or JWT plugin
- **Custom post type** recommended -- keeps AEO pages separate from existing blog/pages
- **AIOSEO Pro REST API** may allow programmatic schema + meta description injection
- **Sitemap:** AIOSEO auto-includes all published posts/pages. AEO pages get picked up automatically if published.
- **Page template:** Divi-compatible. Either use a minimal Divi template or create a custom page template specifically for AEO content.
- **robots.txt:** WordPress generates dynamically via `do_robots` hook, or can be overridden with a physical file. Need to add AI bot allow rules.

### Key Differences from Syntora Pipeline
| Syntora.io | Helping Hands |
|-----------|--------------|
| Next.js 15 (ISR) | WordPress + Divi |
| Supabase (solution_pages table) | WP REST API (posts/pages) |
| Vercel (edge CDN) | A2/H2O (LiteSpeed) |
| Custom sitemap API routes | AIOSEO auto-generated sitemaps |
| IndexNow on publish | Google Search Console API ping |
| Full control over schema in JSX | AIOSEO Pro handles schema |
| Python -> Supabase insert | Python -> WP REST API POST |

### Hosting Considerations
- A2 Hosting (LiteSpeed) can handle page volume. LiteSpeed Cache plugin is already installed.
- If generating 100 pages/day, total AEO pages could reach 3,000+ in a month. Need to verify hosting plan supports this volume.
- Consider separate WordPress sitemap for AEO pages (AIOSEO Pro supports custom sitemap groups).
- Page weight should be minimal -- text-only AEO content, no heavy Divi layouts.
