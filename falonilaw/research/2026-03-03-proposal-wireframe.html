Staged Rollout
Build in phases. Each stage stands on its own.

Stage 1: GitHub + Documentation

  • Set up GitHub Organization and repository
  • Import all existing scripts from each Workspace
  • Document current JST setup, data flows, script inventory
  • Map compliance requirements (what code touches debtor data)
  • Configure permissions, branch protection, code ownership rules

Stage 2: Automation Pipeline

  • GitHub Actions: automated linting and validation on every pull request
  • Auto-deploy from main to AWS Workspaces (replaces manual copy)
  • Staging environment for testing before production
  • AWS backups and recovery testing
  • Enable secret scanning and dependency alerts

Stage 3: ETL Pipeline + Audit Trail

  • MaxExtract/MaxUpdater wrapper scripts in the repo
  • Nightly ETL pipeline: extract, validate, load into secure database
  • Append-only audit table for all data access and pipeline runs
  • Compliance gate: Trisha required reviewer on sensitive code paths

Stage 4-5: Future State (When Ready)

  • Advanced security scanning and compliance automation
  • Infrastructure-as-code for new AWS resources
  • Monitoring dashboards and alerting
  • Runbooks and failure simulation

When you hire your next programmer

They clone the repo. They see the folder structure, the scripts, the commit history, who owns what. They open a pull request, Sean reviews it, and they're contributing. No weeks of shadowing. No "ask Sean where the code is."

When the tools mature

GitHub is the platform. As your needs grow, specialized tools plug in one at a time -- advanced security scanning, dedicated monitoring, infrastructure-as-code. Each one connects to GitHub. You never start over.

When you're ready for AI

AI agents need three things: a structured repo to read from, a pipeline to deploy through, and a review process to gate the output. Without GitHub, AI has nowhere to plug in.
